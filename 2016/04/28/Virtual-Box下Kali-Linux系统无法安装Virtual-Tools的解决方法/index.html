Virtual Box下Kali Linux系统无法安装Virtual Tools的解决方法

发表于 2016-04-28   |   分类于 Web安全   |  

背景：Kali Linux 2.0 （32位）

安装Virtual Tools时

  • 复制桌面Virtual Tools光盘里面的“VBoxlinuxAdditions.run”文件到根目录root文件里，将权限改为(读写)
  • 终端输入./VBoxlinuxAdditions.run，运行后出现若干语句，其中提示：

经Google搜索后，原因为缺失Linux 内核头文件

  • 首先更新源，换163或阿里云的源，
  • 然后采用apt-get update && apt-get install -y linux-headers-$(uname -r)命令安装内核头文件，结果提示

正在读取软件包列表… 完成

正在分析软件包的依赖关系树

正在读取状态信息… 完成

E: 未发现软件包 linux-headers-4.3.0-kali1-686-pae

E: 无法按照正则表达式 linux-headers-4.3.0-kali1-686-pae 找到任何软件包

直接apt-get install linux-headers-4.3.0-kali1-686-pae获取文件失败，提示同样错误信息

具体原因是因为当前版本Kali linux的源内不再包含上述头文件

具体解决方案：

于是手动下载内核头文件以及相关依赖包后使用dpkg命令安装上（注意顺序）

然后再次输入./VBoxlinuxAdditions.run后，Virtual Tools成功安装

内核头文件以及相关依赖包下载地址：

http://http.kali.org/kali/pool/main/l/linux/

http://http.kali.org/kali/pool/main/l/linux-tools/

顺带一提，

  • Paraellels Desktop的增强工具解决方案差不多，都是缺少内核头文件导致安装失败

  • 而VMware 则Kali 官方提供了VM tools安装方法。

总结：

百度不靠谱，很多问题解决方法都是重复而且无效的，自己手动一个个剖析问题，找到问题的突破口，然后在Google这些突破点，最终解决了问题。
