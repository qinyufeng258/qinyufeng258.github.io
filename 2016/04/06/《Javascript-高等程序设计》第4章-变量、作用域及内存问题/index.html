《Javascript 高等程序设计》第4章 变量、作用域及内存问题

发表于 2016-04-06   |   分类于 WEB前端   |  

1、基本类型

  • Undefined、Null、Boolean、Number、String
    • 按值访问，值保存在变量中
    • 不能为其添加属性，即便添加了也是undefined

2、引用类型

  • Javascript 不允许直接访问内存中的位置，所以操作对象时是对对象的引用的操作
  • 按引用访问
  • EMCAScript 认为字符串不是引用类型

2.1、动态属性

  • 当复制保存着某个对象的变量时，操作的是对象的引用，但是为对象添加属性时，操作的是对象本身。

2.2、复制变量值

  • 对于基本类型：复制变量的值就是新建一个副本，二者操作互不影响。
    • var num1 = 5; var num2 = num1;
  • 对于引用类型：复制变量的值就是复制对其的引用，二者操作互相影响

2.3、传递参数：

  • ECMAScript 中所有函数的参数都是按值传递的。
  • 对于基本类型参数：与复制基本类型变量一样，互不影响。
  • 对于引用类型参数：与引用类型变量一样，互相影响。
  • 这里有个疑惑，引用类型是按引用传递，但是参数都是按值传递：
    • 传递引用类型的值时，会把值在内存中的地址复制给一个局部变量
    • 传递基本类型的值时，会把值复制给一个局部变量

2.4、检测类型

  • typeof：
    • 检测字符串、数字、布尔值、undefined的工具
    • 检测对象或null，返回都是object
  • instanceof：
    • 检测引用类型，根据原型链识别

3、执行环境与作用域

全局执行环境是最外围的一个执行环境，根据ECMAScript实现所在的宿主环境不同，执行环境的对象也不一样。在WEB浏览器中，全局执行环境是window对象。

当代码在一个环境中执行时，会创建对象的一个作用域链。其作用是保证对执行环境有权访问的所有变量和函数的有序访问。作用域链的前端是当前执行代码所在环境的变量对象。如果环境是函数，则变量对象就是其活动对象，初始时只包含arguments对象。

4、垃圾收集

JavaScript有自动垃圾回收机制：找出那些不再继续使用的变量，然后释放占用的内存。
函数中局部变量的正常生命周期：周期内，会为局部变量在栈（堆）内存上分配相应的空间，周期结束后释放内存

  • 标记清除策略
    • 变量进入环境，将变量标记为“进入环境”，离开环境，标记为“离开环境”
    • 具体策略：垃圾收集器在运行的时候会给存储在内存里的变量上标记，然后它会去掉环境中的变量以及被环境中的变量引用的变量的标记，此后再被加上标记的变量视为准备删除的元素，最后收集器完成内存清除工作。

      IE\Chrome\Opera\Firefox\Safari 的JavaScript都是使用标记清除策略。

  • 引用计数
    • 记录每个值被引用的次数，当次数为0时，释放内存。
    • 对于JavaScript来说很少见
  • 性能问题
    • IE7 以前是根据内存分配量来运行垃圾收集器的，到达某个极限就自动收集，非常shit
    • IE7 重写了该过程，改为动态修正。
  • 管理内存
    • 解除一个值的引用并不意味着自动回收该值所占内存，而是让该值脱离执行环境，以便垃圾收集器瑕疵运行时能够回收。
