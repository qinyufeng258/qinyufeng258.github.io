Ajax：Asynchronous JavaScript And XML ，其核心是XMLHttpRequest对象（XHR）
注：Ajax通信与数据格式无关，不一定是XML数据

1、XMLHttpRequest 1级

1.1、XHR用法：

  • 创建XHR：var xhr = new XMLHttpRequest();
  • 启动一个请求：xhr.open(“get”, “index.php”, false);
    • 参数为：发送请求类型、请求的URL、是否异步发送请求
    • 不会发送请求，而是准备发送
  • 发送请求：xhr.send();
    • 参数为：请求主体的数据，没有需传入null
  • 响应数据：响应数据会返回给xhr对象，属性为：
    • responseText：响应主体文本
    • responseXML：若响应内容类型是“text/xml”或“application/xml”，则包含响应数据的XML DOM 文档
    • status：响应的HTTP状态
    • statusText：HTTP状态说明
  • 同步请求：检查响应状态并做相应处理
  • 异步请求：监听并处理
    • readyState属性：表示请求/响应过程阶段
      • 0：未初始化
      • 1：启动
      • 2：发送
      • 3：接受
      • 4：完成
    • 根据readyState做处理：跨浏览器兼容性XHR请求——响应
    • xhr.abort()：在接受以前，可以终止异步请求

3、进度事件

  • loadstart：在接受响应数据的第一个字节时触发
  • progress：在接受响应期间不断触发
  • error：在请求发生错误时触发
  • abort：在调用abort()终止请求时触发
  • load：收到完整响应数据时触发
  • loadend：通信完成或error、abort、load后触发

支持浏览器：F 3.5+、S 4+、Chrome、IOS的Safari、Android的Webkit、Opera、IE8支持load
没有浏览器支持loadend

3.1、load事件

  • Firefox、Opera、Chrome、Safari 支持load事件
  • 与readstatechange用法一致。

4、跨域资源共享

CORS (跨域资源共享），定义了在必须访问跨域资源时，浏览器与服务器该如何通信。
基本思想就是使用自定义的头部信息让浏览器与服务器进行沟通
一个简单的GET\POST请求，没有自定义头部，附加一个额外的Origin头部（协议、域名与端口），服务器可以根据头部信息来决定是否给予响应。
Origin：http://www.lalala.net
如果服务器认为可以接受，就在Access-Control-Allow-Origin头部中发送相同源信息。

4.1、IE中的CORS

  • XDR（XDomainRequest）
    • cookie不会发送和返回
    • 只能设置请求头部的content-type字段
    • 不能访问响应头部
    • 只支持GET和POST
    • 支持timeout属性和ontimeout事件处理程序

4.2、其他浏览器的CORS

  • 与普通的XHR相同，URL为绝对URL
  • 限制
    • 不能自定义头部
    • cookie不会发送和返回
    • getAllResponseHeaders 总是返回空字符串

5、一些好玩的跨域技术

  • 图片ping

    • 常用于跟踪用户点击页面或动态广告曝光次数

  • JSONP （JSON with Padding）

    • 回调函数就是响应到来时应该在页面上调用的函数
    • 数据就是回调函数传入的JSON数据

  • Comet （服务器向页面推送数据）

    • 长轮询：页面发送一个到服务器的请求，然后服务器一直保持连接打开

    • 流：在整个页面的生命周期内只使用一个HTTP连接

  • WebSockets
  • SSE
